# Итоговое задание: Автоматизированный мониторинг и реагирование на угрозы
## Описание проекта
Данный проект представляет собой инструмент на Python для автоматизированного сбора данных об угрозах, их анализа и имитации реагирования. Скрипт анализирует сетевые события и проверяет актуальные уязвимости, формируя итоговую отчетность с визуализацией.
## Источники данных
В соответствии с требованиями задания, используются два источника данных:
1. **Логи Suricata** — для анализа событий сетевой безопасности и выявления подозрительных IP-адресов  (данные генерируются локально для стабильности тестирования).
2. **API Vulners** — для поиска опасных уязвимостей (реализована имитация ответа API для работы без ключей авторизации).
## Логика работы скрипта
1. **Сбор данных:** Загрузка логов сетевого трафика и получение списка уязвимостей.
2. **Анализ:** Использование библиотеки `pandas` для подсчета частоты событий от различных IP-адресов и фильтрации уязвимостей по высокому баллу CVSS (>= 7.0).
3.**Реагирование:** Вывод в консоль предупреждений о найденных угрозах, имитация блокировки подозрительных IP-адресов и отправки уведомлений администратору.
4.**Отчетность:** Сохранение результатов анализа частоты событий в файл формата `.csv` и построение столбчатой диаграммы (сохраняется в `.png`) с помощью `matplotlib`.

## Состав репозитория
* `main.py` — основной исполняемый скрипт.
* `threat_report.csv` — сформированный отчет по подозрительным IP-адресам
* `threat_plot.png` — визуализация распределения событий (график).
* `suricata_logs.json` — локальный файл с логами, который скрипт создает для анализа.
